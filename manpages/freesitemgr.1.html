NAME

freesitemgr - Insert or re-insert freesites into freenet

 

SYNTAX

freesitemgr [options] command [arguments]  

DESCRIPTION

freesitemgr is a simple command-line freenet client program for inserting freesites.

Use of this program is controlled by the command argument you give it.

run freesitemgr -h for a detailed help output.  

OPTIONS

-h, --help
Print help information and exit
-v, --verbose Enable very verbose output, including FCP transcripts, to the stderr stream.
-q, --quiet Run quietly
-l, --logfile location of logfile (default /home/david/updatesites.log)
-s, --single-files Insert each file of the freesite as single CHKs, and redirect to these from within the freesite manifest. You must use this option if inserting through an FCP port not on your local machine. Note that files will be inserted one at a time, with each next file not being inserted till the previous one is completely inserted. If you wish to insert more than one file concurrently, use the -a or -m options.
-a, --all-at-once Enables concurrent insert of the site's files. By default, a maximum of 10 files will be inserting at any one time. You can increase this with -m. Note that this -a option automatically turns on -s.
-m, --max-concurrent-inserts=num Turns on -s and -a, and sets the maximum number of concurrent file inserts to num. You can set a very high value if you want, but it will be of little use because the node itself regulates how many concurrent inserts it performs.
-i, --insert-all Forces insert of all files in all aites. Otherwise, sites with no new or changed files will be skipped, and in sites that have changed, only the new/changed files (and a fresh manifest) will be inserted.

 

COMMANDS

setup Create or modify the freesite config file (~/.freesites) interactively.

add Add a new freesite interactively

list [<name>] display a summary of all freesites, or a detailed report of one site if <name> given

remove <name> remove given freesite

update reinsert any freesites which have changed since they were last inserted

 

ENVIRONMENT VARIABLES

FCP_HOST
Specifies the hostname of the FCP interface. This value will be used unless overridden with '-H'.
FCP_PORT
Specifies the port number of the FCP interface. This value will be used unless overridden with '-P'.
